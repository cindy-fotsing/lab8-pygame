Lab 8 Architecture

A single-file Pygame app that animates a fixed population of squares with random motion, edge bouncing, threat-seeking behavior, and automatic respawn.

Sequence Diagram

sequenceDiagram participant U as "User" participant M as "main()" participant P as "Pygame" participant E as "Event Queue" participant S as "Square" participant R as "Random / Math" U->>M: Launch program M->>P: init, create window, create clock M->>S: Create 20 Square instances loop Each frame at 60 FPS M->>E: Poll events alt Quit event received E-->>M: pygame.QUIT M->>P: quit M->>U: Exit program else Continue running M->>P: Fill background loop For each square M->>S: update(squares) S->>R: compute center, distance, and random fallback alt A larger square is nearby S->>R: steer velocity toward threat else No threat nearby S->>R: occasionally randomize direction end S->>S: move and bounce at window edges end M->>M: remove expired squares alt Population below target M->>S: spawn new Square instances end loop For each square M->>S: draw(surface) S->>P: draw rectangle end M->>P: flip display and cap frame rate end end

Notes

  • The runtime uses SQUARE_COUNT = 20, even though the README still mentions 10 squares.
  • Square.update() combines motion, bouncing, threat-seeking, and lifespan decay in one step.
  • The app is intentionally self-contained: there are no extra project modules beyond the main entry point and dependency file.

All diagrams are based on traced behavior in main.py.
